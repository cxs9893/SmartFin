# SmartFin 下一迭代任务清单（基于 A-D 迭代文档）

## 说明
- 本清单基于以下文档中的“已知风险/限制”与“建议下一迭代”提炼：
  - `docs/ingest-iteration.md`
  - `docs/retrieval-iteration.md`
  - `docs/qa-iteration.md`
  - `docs/report-iteration.md`
- 任务按模块拆分，并补充跨模块协作项与完成定义（DoD）。

## A 模块（Ingest / Indexing）
### 目标
- 降低脏数据导致的证据质量问题，并为下游提供更稳定、可评估的索引能力。

### 任务
1. 引入可替换 embedding 提供器（本地/云端可切换）
- 优先级：P0
- 交付：统一 embedding 接口 + 至少 2 个实现（本地哈希/真实模型）
- 验收：同一数据集可切换 provider 并成功构建索引

2. 补充英文清洗与字段完整性规则
- 优先级：P0
- 交付：`text/quote_en` 清洗策略与异常数据降级处理
- 验收：新增测试覆盖混合语种、异常编码、缺字段场景

3. 产物元数据增强（供 report 聚合）
- 优先级：P1
- 交付：补充章节归一化标签、年份解析稳定性
- 验收：report 场景下噪声段落命中率下降（对比基线）

## B 模块（Retrieval）
### 目标
- 提升检索语义质量与可解释性，形成“可调优、可评估、可观测”的检索链路。

### 任务
1. 落地持久化向量索引读取链路（与 indexing 对齐）
- 优先级：P0
- 交付：从构建产物加载向量索引，避免查询时实时构建向量
- 验收：查询耗时下降，检索结果与旧链路一致或更优

2. 建立检索评估基线（Recall@K / MRR）
- 优先级：P0
- 交付：评估脚本 + 小规模标注集 + 指标报告模板
- 验收：可一键输出评估结果并纳入迭代文档

3. 增加融合排序可观测日志
- 优先级：P1
- 交付：候选截断、归一化分数、融合得分日志字段
- 验收：线上问题可回溯到具体候选与打分过程

4. 给 QA 输出判定信号
- 优先级：P1
- 交付：`hybrid_search` 结果补充最小分数阈值、候选分布等信号
- 验收：QA 可基于信号稳定实现“回答/拒答”门控

## C 模块（QA）
### 目标
- 在 grounded 基础上提升答案稳健性，减少偏题与冲突证据导致的误答。

### 任务
1. 增加 evidence-query 相关性过滤
- 优先级：P0
- 交付：证据筛选规则（阈值或重排）与拒答联动
- 验收：偏题问题误答率下降（对比当前基线）

2. 多证据冲突检测与置信度衰减
- 优先级：P0
- 交付：冲突检测策略 + `confidence` 调整逻辑
- 验收：冲突场景测试通过，输出解释清晰

3. 强化英文证据判定
- 优先级：P1
- 交付：从“字符级判定”升级为更稳健的语言判定策略
- 验收：混合语种与噪声编码场景测试通过

4. 与 report 统一 citation 展示契约
- 优先级：P1
- 交付：字段命名、截断策略、排序规则统一文档与实现
- 验收：`ask/report` 输出可由同一前端渲染器消费

## D 模块（Report / Docker / Docs）
### 目标
- 提升报告业务准确性与交付稳定性，降低环境因素导致的验收波动。

### 任务
1. 引入固定数据夹具并补齐报告业务断言
- 优先级：P0
- 交付：`single_year/cross_year` 业务断言测试样本
- 验收：关键字段与聚合逻辑可稳定回归

2. CI 增加容器冒烟阶段
- 优先级：P0
- 交付：Docker daemon 可用时段执行 `docker-compose up --build` 的流水线步骤
- 验收：产物文件与容器状态自动检查

3. 降低报告噪声段命中（协作 ingest/retrieval）
- 优先级：P1
- 交付：报告查询模板优化 + 章节去噪规则
- 验收：`Signature` 等无效高频段落占比下降

## 跨模块协作任务
1. 统一 `hit/citation/evidence` 契约版本
- 优先级：P0
- 交付：契约文档 + 类型定义 + 回归测试
- 影响模块：A/B/C/D

2. 固化“迭代文档 + 测试报告”自动检查
- 优先级：P0
- 交付：在 CI 执行 `scripts/validate_iteration_docs.ps1` 与测试 Skill 输出检查
- 影响模块：A/B/C/D/docs

3. 建立端到端联调流水线（ingest -> retrieval -> qa/report）
- 优先级：P1
- 交付：一条可复现的全链路命令与结果快照
- 影响模块：A/B/C/D

## 建议排期（两周）
1. 第 1 周（P0）
- A-1、A-2、B-1、B-2、C-1、D-1、跨模块-1、跨模块-2

2. 第 2 周（P1）
- A-3、B-3、B-4、C-2、C-3、C-4、D-2、D-3、跨模块-3

## 完成定义（DoD）
- 每个任务至少包含：
  - 代码改动（或文档/脚本改动）
  - 对应测试与命令结果
  - 迭代文档回填（`Validation and Results` + 风险）
- 合并前必须满足：
  - `test_smoke` 通过
  - 模块专项测试通过
  - 验收映射标注 `PASS/FAIL/N/A` 且有证据
